Hack Club Identity Integration

Purpose: Verify user identity with official Hack Club records and collect address information

Flow: During OAuth consent, users are required to verify their identity through Hack Club's identity service

Benefits: Ensures users are authentic Hack Club members with verified identities and provides access to address information

Status Values:

  • verified - Identity successfully verified, address information available
  • pending - Verification under review
  • rejected - Verification rejected
  • unverified - No verification attempted

Address Information: When identity verification is completed, address details (street, city, state, postal code, country) become available through the /oauth/user endpoint

Integration: Works seamlessly with OAuth flow when "Verify Identity" checkbox is selected. Address scope is automatically included.

Understanding Authentication Types

Choose the right authentication method for your use case

OAuth Access Tokens

For user-facing applications
User-specific data access
Limited to user's own data
Requires user consent
Expires after 1 hour (refreshable)
Scoped permissions (clubs:read, users:read, etc.)
Endpoints: /oauth/user, /oauth/user/clubs, /oauth/user/projects
Use case: Personal apps, user dashboards

Admin API Keys

For platform administration
Platform-wide access
Access to all clubs, users, analytics
Search/lookup functionality
No expiration (until revoked)
Endpoints: /api/v1/clubs, /api/v1/users, /api/v1/analytics
Use case: Admin tools, monitoring, search

Quick Decision Guide

Use OAuth when:
  • Building user-facing apps
  • Users access their own data
  • Need user consent/login
Use Admin API when:
  • Need to search users/clubs
  • Building admin dashboards
  • Need platform-wide data
